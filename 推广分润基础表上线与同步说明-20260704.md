# 推广分润基础表上线与同步说明

## 本次新增内容

- 本次一次性新增 3 张基础表：
  - `promotion_relations`
  - `commission_rules`
  - `commission_split_logs`
- 对应迁移文件：
  - `backend/migrations/114_add_promotion_foundation_tables.sql`
- 目的：
  - 为后续“代理 -> 分销 -> 用户”的补差法分润打底
  - 先把基础关系表、规则表、分润明细表一次性建齐
  - 后续即使先不用规则表，也不用反复升级数据库

## 三张表职责

### 1. promotion_relations

- 作用：
  - 固定用户归属到哪个代理、哪个分销
  - 防止后续调整上下级关系后，历史订单分润串账
- 主要字段：
  - `user_id`
  - `agent_user_id`
  - `distributor_user_id`
  - `direct_parent_user_id`
  - `direct_parent_role`
  - `binding_source`
  - `is_locked`
  - `bound_at`

### 2. commission_rules

- 作用：
  - 存平台默认规则
  - 存代理覆盖规则
  - 存分销覆盖规则
  - 支持补差法和叠加法扩展
- 主要字段：
  - `calc_mode`
  - `agent_target_rate`
  - `distributor_target_rate`
  - `freeze_hours`
  - `scope_type`
  - `scope_id`
  - `priority`
  - `effective_at`
  - `expired_at`

### 3. commission_split_logs

- 作用：
  - 一笔订单拆成多笔收益明细
  - 记录分销拿多少、代理补差拿多少
  - 便于后续做待结算、已结算、退款冲正
- 主要字段：
  - `order_id`
  - `consumer_user_id`
  - `beneficiary_user_id`
  - `beneficiary_role`
  - `agent_user_id`
  - `distributor_user_id`
  - `level`
  - `calc_mode`
  - `base_amount`
  - `target_rate`
  - `parent_rate`
  - `commission_amount`
  - `status`
  - `rule_id`
  - `relation_snapshot`

## 本地开发同步步骤

### 1. 拉最新代码

```bash
git pull
```

### 2. 进入后端目录

```bash
cd backend
```

### 3. 重新生成 ent 相关代码

```bash
go generate ./ent
go generate ./cmd/server
```

### 4. 启动后端

```bash
go run ./cmd/server
```

### 5. 启动时自动执行迁移

- 新版本后端启动后，会自动执行：
  - `114_add_promotion_foundation_tables.sql`
- 执行成功后，`schema_migrations` 会新增对应记录

## 线上服务器同步步骤

### 场景一：源码直接部署

```bash
cd /your/path/subiohub
git pull
cd backend
go generate ./ent
go generate ./cmd/server
go build -o subiohub-server ./cmd/server
```

- 然后重启你的服务：

```bash
systemctl restart subiohub
```

- 如果你不是 `systemd`，就换成你当前自己的启动方式

### 场景二：Docker / 容器部署

- 拉最新代码
- 重新构建镜像
- 推送镜像
- 先起一台新容器完成 migration
- 确认三张表创建成功后，再滚动替换其他实例

## 上线前必须做的事

- 备份数据库
- 不要修改旧 migration
- 只新增新的 migration 文件
- 确认线上运行的是新构建出来的后端，不是旧二进制

## 上线后检查

### 1. 检查迁移记录

```sql
SELECT filename, applied_at
FROM schema_migrations
WHERE filename = '114_add_promotion_foundation_tables.sql';
```

### 2. 检查三张表是否存在

```sql
SELECT table_name
FROM information_schema.tables
WHERE table_schema = 'public'
  AND table_name IN (
    'promotion_relations',
    'commission_rules',
    'commission_split_logs'
  );
```

### 3. 检查关键索引是否存在

```sql
SELECT indexname
FROM pg_indexes
WHERE schemaname = 'public'
  AND tablename IN (
    'promotion_relations',
    'commission_rules',
    'commission_split_logs'
  );
```
