# 渠道管理使用说明

## 1. 渠道是什么

在 `subiohub` 里，`渠道` 不是单纯的展示分类，而是一个面向业务的“请求入口配置”。

一个渠道通常同时决定下面几件事：

- 请求可以走哪些平台和分组
- 请求中的模型名是否需要映射
- 计费时应当使用哪个模型名作为基准
- 这个请求应该按什么规则定价
- 该定价是否同步影响账号统计

可以把它理解成：

- `分组`：底层资源池
- `渠道`：面向业务和客户端的对外出口
- `模型映射`：名称转换层
- `模型定价`：计费层

## 2. 一个请求经过渠道时会发生什么

当客户端发起一次模型请求时，系统大致会按下面顺序处理：

1. 根据渠道找到可用的平台和分组
2. 判断请求模型名是否需要映射到上游模型名
3. 决定计费时要用哪个模型名查价
4. 按渠道中的定价规则计算成本或售价
5. 将请求发往对应平台下的账号池

所以，渠道本质上是在定义一条：

`路由规则 + 模型映射规则 + 计费规则`

## 3. 页面字段分别做什么

### 3.1 名称 / 描述

用于管理员区分不同渠道。

典型场景：

- 官网默认渠道
- 企业客户专用渠道
- OpenAI 兼容渠道
- 测试渠道

### 3.2 平台配置

用于决定这个渠道允许使用哪些平台：

- `Anthropic`
- `OpenAI`
- `Gemini`
- `Antigravity`

这一步只是确定“平台范围”，真正使用哪些资源，还要看下面的分组绑定。

### 3.3 平台分组绑定

把渠道和具体分组关联起来。

例如：

- 某个渠道只允许走 `Anthropic` 下的 `默认组`
- 另一个渠道可以同时走 `OpenAI` 和 `Gemini` 的多个分组

这个配置的作用是把“业务入口”和“底层资源池”连接起来。

### 3.4 限制模型

开启后，只有你在“模型定价”里明确配置过的模型才能被这个渠道使用。

用途：

- 防止客户端传入未配置模型
- 防止请求穿透到不受控的上游模型
- 保证计费和路由都只发生在明确维护过的模型范围内

### 3.5 计费基准

它决定计费时到底用哪个模型名去查价。

当前支持：

- `以渠道映射后的模型计费`
- `以请求模型计费`
- `以上游最终模型计费`

适用场景：

- 如果你希望用户看到什么模型名，就按那个名字收费，用“请求模型”
- 如果你希望内部统一口径，按映射后名字收费，用“映射后模型”
- 如果上游可能再次改写模型名，需要最终落地一致，就按“上游最终模型”

### 3.6 模型映射

作用是把客户端传入的模型名映射成上游实际使用的模型名。

典型用途：

- 兼容旧模型名
- 给外部提供统一别名
- 内部切换模型版本而不影响客户端调用

示例：

```text
请求模型: claude-3.5-sonnet
映射后:  claude-sonnet-4-20250514
```

这样做的好处是，客户端仍然用熟悉的名称调用，但后台可以灵活切换真正使用的上游模型。

### 3.7 模型定价

用于定义该渠道下不同模型怎么收费。

当前已支持的模式：

- `Token`
- `按次`
- `图片按次`

常见用途：

- 对不同模型设置不同售价
- 把外部售价和上游成本区分开
- 根据不同业务渠道做差异化计费

例如：

- 企业渠道价格更高
- 测试渠道价格更低
- 某些内部渠道甚至可以配置为 0 成本

### 3.8 将定价应用到账号统计

这是一个统计口径相关开关。

开启后，渠道定价规则会同时影响账号层面的统计数据。

适合场景：

- 希望账号分析报表和渠道售价口径一致
- 需要从账号视角反推渠道收益

如果你只是想控制用户侧计费，不一定必须开启。

## 4. 一个实际配置例子

### 场景：做一个 Claude 兼容渠道

目标：

- 对外暴露旧模型名
- 内部切到新版 Sonnet
- 只走 `Anthropic` 平台资源
- 按你自己定义的价格收费

可以这样配置：

#### 基础配置

- 名称：`Claude 默认渠道`
- 平台：`Anthropic`
- 分组：绑定 `Anthropic` 平台下的默认分组
- 限制模型：开启
- 计费基准：`以渠道映射后的模型计费`

#### 模型映射

```text
claude-3.5-sonnet -> claude-sonnet-4-20250514
claude-3-opus     -> claude-opus-4-20250514
```

#### 模型定价

```text
平台: Anthropic
模型: claude-sonnet-4-20250514
模式: Token
输入价: 3
输出价: 15
```

这样，客户端依旧发：

```text
claude-3.5-sonnet
```

系统内部会：

1. 映射到 `claude-sonnet-4-20250514`
2. 路由到 Anthropic 对应分组
3. 按配置好的价格结算

## 5. 当前已经完成的能力

当前 `next-web` 渠道管理已支持：

- 搜索
- 状态筛选
- 状态切换
- 创建 / 编辑 / 删除渠道
- 平台选择
- 平台分组绑定
- 模型映射编辑
- 模型定价编辑

## 6. 后续计划

下面两项已经明确纳入后续实现计划：

- `区间定价编辑`
- `账号统计定价规则`

说明：

- `区间定价编辑`：用于按上下文 token 区间或层级定义更细粒度的价格
- `账号统计定价规则`：用于让账号统计采用独立于渠道默认定价的规则

这两项属于渠道管理的增强能力，当前文档和开发计划中已标记为未来实现内容。
