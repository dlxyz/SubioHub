# SubioHub Web 迭代计划

## 1. 当前目标

项目当前的 Web 端统一由 `next-web` 承接。

本文档用于记录当前 Web 侧的演进方向，确保公开站、用户中心、管理后台、SEO、多语言与内容模块在同一条技术主线上推进。

## 2. 当前技术基线

- Web 框架：Next.js App Router
- UI：React + TypeScript + Tailwind CSS
- 状态管理：Zustand
- 多语言：路由级 i18n + 本地语言包
- 后端协作：Go API

## 3. 已完成能力

### 3.1 公开站

- 公开页面已按 `next-web` 作为正式交付入口
- 支持 SSR / SSG 所需的 metadata、canonical、alternates 与 sitemap 基础能力
- 后续资讯、帮助中心、价格页等内容页都继续沿用当前站点架构

### 3.2 用户中心

- 已具备用户中心布局、顶部导航、侧边导航和基础占位页
- 已接入语言切换、本地持久化与页面 `lang` 同步
- 已覆盖仪表盘、API 密钥、调用日志、钱包充值、邀请分销、个人设置等核心页面

### 3.3 管理后台

- 已具备完整后台布局和导航骨架
- 已接入管理总览、运维监控、用户管理、分组管理、账号管理、公告管理、系统设置、分销管理、订单管理、支付看板、套餐管理、折扣码管理、调用记录、代理管理、卡密管理等模块
- 运维监控已落为真实业务页，不再停留在占位路由

### 3.4 多语言

- 已建立 `zh-CN` / `en-US` 的语言包组织方式
- 用户端与管理端共用统一语言切换机制
- 后续新增语言时继续扩展现有 `next-web` i18n 结构

## 4. 继续推进顺序

### 4.1 内容与资讯

- 接入资讯列表、资讯详情、多语言发布与 SEO 输出
- 让资讯内容与公开站现有路由级国际化保持一致

### 4.2 管理后台增强

- 继续完善账号管理的复杂表单与平台差异逻辑
- 继续补全运维监控中的告警规则、告警事件、错误详情与实时刷新
- 持续提升系统设置、订单、支付、订阅等模块的多语言完整度

### 4.3 Web 体验优化

- 继续优化移动端响应式布局
- 收敛重复组件和表格交互模式
- 按页面重要性补充更精细的 SSR / SEO 能力

## 5. 开发原则

- 新页面与新功能统一落在 `next-web`
- 先补稳定 API，再补页面联调
- 文档描述统一以当前 Web 架构为准
- 多语言、SEO、响应式和后台可用性同步考虑，不再拆成两套口径
