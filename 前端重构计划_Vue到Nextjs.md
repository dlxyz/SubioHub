# SubioHub 前端重构计划 (Vue 3 -> Next.js)

## 1. 背景与目标
当前 `frontend` 目录使用的是基于 Vue 3 + Vite + SPA (Single Page Application) 的技术栈。
虽然开发体验好且交互流畅，但纯 SPA 架构在 SEO（搜索引擎优化）和首屏加载时间（FCP）上存在天然劣势。
为了更好地进行产品推广、内容收录以及提供更现代化的全栈开发体验，我们决定**使用 Next.js (React) 将前端彻底重构，并最终废弃 Vue 前端**。

## 2. 现状评估 (现有 Vue 项目规模)
通过分析现有的 `frontend/src` 目录，前端项目体量中等偏大，包含以下核心模块：

### 2.1 文件规模统计 (估算)
- **Views (页面/路由级组件)**: 约 35+ 个 (`src/views/`)
  - 管理端 (Admin): 包含仪表盘、用户管理、渠道、账单、代理、网关设置等。
  - 用户端 (User): 包含控制台、密钥管理、使用统计、订阅与充值、分销等。
  - 公共/认证端 (Auth): 登录、注册、OAuth 回调、初始化 Setup。
- **Components (UI 组件)**: 约 80+ 个 (`src/components/`)
  - 大量细分的表格、弹窗、图表 (基于 Chart.js)、支付模块。
- **API 封装**: 约 30+ 个服务封装文件 (`src/api/`)
- **Composables (Vue Hooks)**: 约 15+ 个 (`src/composables/`)
- **Store (状态管理)**: 基于 Pinia 的状态树 8 个 (`src/stores/`)
- **其他**: 包含 `i18n` 多语言、`utils` 工具函数、路由守卫等。

### 2.2 核心依赖清单
- UI 框架: Vue 3 + Tailwind CSS
- 路由与状态: Vue Router + Pinia
- 图表库: Chart.js (`vue-chartjs`)
- 请求库: Axios
- 其他: Stripe JS, Marked (Markdown), Vue-i18n

## 3. 重构技术选型 (Next.js Stack)

为保证重构后的系统既具备 SEO 能力，又能保持优秀的 UI 体验和后续维护性，建议采用以下技术栈：

1. **核心框架**: **Next.js 14+ (App Router)**
   - 充分利用 SSR (服务端渲染) 和 SSG (静态生成) 解决 SEO 问题。
   - 使用 Server Components 减少客户端 JS 体积。
2. **样式与 UI 组件**: **Tailwind CSS + shadcn/ui (或 Radix UI)**
   - 完美平替现有的纯 Tailwind 写法，并提供无头 (Headless) 且美观的交互组件。
3. **状态管理**: **Zustand** (轻量) 或 **React Context**
   - 平替 Pinia。
4. **数据请求与缓存**: **SWR** 或 **TanStack Query (React Query)** + 原生 `fetch`
   - 取代 Axios + 手写加载状态，更好地与 Next.js 服务端获取融合。
5. **图表库**: **Recharts** 或继续使用 **Chart.js (react-chartjs-2)**
6. **多语言**: **next-intl**
   - 当前 `next-web` 已先落地一套轻量 i18n 方案：`zustand locale store + messages/locales + useI18n()`。
   - 后续如果语言种类显著增加，再评估是否切换到 `next-intl`，避免在迁移中途引入过重的国际化改造成本。

## 4. 执行操作与分步实施计划

由于项目规模较大，直接“推倒重来”风险极高。重构需采用**“并行开发，逐步替换，最终删除”**的渐进式策略。

### 阶段一：基建与架子搭建 (1-2 天)
1. 在项目根目录（或新建 `next-web` 目录）初始化 Next.js 项目：
   ```bash
   npx create-next-app@latest next-web --typescript --tailwind --eslint --app
   ```
2. 搭建基础架构：
   - 配置 Tailwind 主题颜色（迁移原有配色方案）。
   - 配置 Axios/Fetch 请求拦截器（处理 JWT Token 和 401 跳转）。
   - 搭建 Zustand 状态管理基础（如 AuthStore）。
   - 配置 i18n 基础目录。

### 阶段二：公共与 SEO 核心页面重构 (3-4 天)
这些页面对 SEO 要求最高，优先重构并享受 SSR 红利：
1. **官网首页/落地页 (Home)**
2. **登录与注册页 (Auth)**
   - 包含普通账密登录、注册表单、忘记密码。
   - 各种 OAuth 回调页面的 React 化。
3. **公告详情页 (Announcements)**

### 阶段三：用户控制台重构 (1-2 周)
用户后台对 SEO 要求不高，但交互复杂：
1. 基础布局：实现侧边栏 (Sidebar) 和顶部导航 (AppHeader)。
2. **仪表盘 (Dashboard)**：迁移各类统计图表。
3. **API 密钥管理 (Keys)**：列表、创建弹窗、复制等交互。
4. **使用统计 (Usage)**：数据表格与明细。
5. **财务模块**：余额、充值支付 (Stripe 对接)、卡密兑换。
6. **分销系统 (Affiliate)**：我们刚完成的新功能面板。

管理后台交互极其繁琐，是整个重构最耗时的部分：
1. 各种资源的数据表格 (Data Table) 封装，包含分页、排序、搜索。
2. 用户管理、渠道配置、代理设置等数十个 CRUD 表单弹窗的 React 重写。
3. Ops 监控大盘与日志展示模块的迁移。

### 阶段四补充：先迁移导航骨架，再逐页填充
为了避免 Next.js 管理后台和原 Vue 管理后台在信息架构上持续跑偏，管理端迁移不再按“想到哪个页就补哪个页”的方式推进，而是先把左侧导航整体对齐，再按导航顺序逐页落功能。

当前确定的执行策略如下：
1. **以 `frontend/src/components/layout/AppSidebar.vue` 为导航基准**：
   - 管理端主菜单顺序对齐 Vue 版本，包含：管理总览、运维监控、用户管理、分组管理、渠道管理、订阅管理、账号管理、公告管理、代理管理、卡密管理、折扣码管理、订单中心、调用记录、系统设置。
2. **保留订单中心的子菜单结构**：
   - `订单中心` 下继续拆分为 `支付看板`、`订单管理`、`套餐管理`，保持与 Vue 后台一致。
3. **插入二开新增菜单**：
   - 在对齐原导航的基础上，新增 `分销管理` 入口，作为本轮二开的后台主线模块。
4. **未迁移页面先占位**：
   - 在 `next-web` 中，导航先全部挂上；尚未完成的页面统一进入占位路由，避免菜单缺口影响联调和任务排期。
5. **后续开发方式改为“按导航清单推进”**：
   - 优先顺序按导航和业务验收链路综合排序，避免遗漏高频后台页。

## 5. 核心原则：全量双端适配 (PC + 手机 H5)

本次重构**必须强制执行“移动端优先 (Mobile First)”与双端响应式开发原则**。
不仅是官网首页和用户端，**哪怕是管理后台 (Admin Dashboard) 的所有页面，也必须保证在手机 H5 端能够正常浏览和操作**。

具体适配要求如下：
1. **布局响应式 (Layout)**：
   - PC 端采用侧边栏 (Sidebar) 布局。
   - H5 端侧边栏必须自动收起，改用顶部汉堡菜单 (Hamburger Menu) 或底部导航栏 (Bottom Navigation) 唤出。
2. **组件响应式 (Components)**：
   - 数据表格 (Data Table) 在手机端需自动转化为卡片列表 (Card List) 或支持横向滚动 (Overflow-X)。
   - 所有的弹窗 (Dialog/Modal) 在手机端自动撑满全屏 (Full-screen sheet) 或变为底部抽屉 (Bottom Sheet)。
3. **Tailwind 实践**：
   - 大量使用 `sm:`, `md:`, `lg:` 等断点前缀。例如 `flex-col md:flex-row` 确保手机端纵向排列，电脑端横向排列。

## 6. 开发建议与注意事项
- **不要直接翻译代码**：Vue 和 React 的思维模式不同。不要试图把 Vue 的 `ref`/`computed` 一比一硬翻成 `useState`/`useMemo`。特别是表单处理和组件生命周期，应采用 React/Next.js 的最佳实践。
- **Server vs Client Components**：在 Next.js App Router 中，默认都是服务端组件。凡是涉及 `onClick`、`useState`、`window` 对象的地方，务必在文件顶部加上 `'use client'` 指令。
- **路由迁移映射**：
  - Vue 的 `vue-router` 配置需要转化为 Next.js 的文件系统路由。
  - 例如 `src/views/user/KeysView.vue` 对应 Next.js 中的 `app/(user)/keys/page.tsx`。
