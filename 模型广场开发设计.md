# 模型广场开发设计

## 1. 目标

本轮先完成前端首页导航与模型广场的第一阶段重构，重点解决以下问题：

- 首页顶部导航中的“模型广场”当前还是占位链接，无法进入真实页面。
- 控制台中的 `/dashboard/models` 仍是占位内容，没有模型筛选、价格展示和能力标签。
- 前端缺少一个可直接渲染模型广场的聚合接口，现有后端接口偏管理端或按单模型查询，不适合首屏批量渲染。

第一阶段目标：

1. 新增公开模型广场接口，返回模型列表、平台信息、能力标签、价格信息和站点文档链接。
2. 新增公开页面 `/models`，用于首页导航访问。
3. 将控制台 `/dashboard/models` 改为复用同一套模型广场主体内容。
4. 首页导航改为跳转真实模型广场页面。

## 2. 页面范围

### 2.1 公开模型广场页

路由：`/models`

页面模块：

- 顶部导航
- 英雄区标题与说明
- 平台筛选 Tabs
- 关键词搜索
- 能力标签筛选
- 模型卡片网格
- 空状态 / 加载状态 / 错误状态

### 2.2 控制台模型广场页

路由：`/dashboard/models`

页面策略：

- 直接复用模型广场主体组件
- 保留控制台现有侧边栏和顶部导航
- 去掉公开页的站点级 header/footer，只保留内容区

## 3. 接口设计

### 3.1 接口路径

`GET /api/v1/settings/public/model-plaza`

原因：

- 当前公开设置已经挂在 `/settings/public`
- 模型广场也属于无需登录即可查看的公开内容
- 便于后续结合站点名称、文档地址等公开配置一起扩展

### 3.3 数据来源

- 模型列表：
  - 优先使用 `GatewayService.GetAvailableModels(ctx, nil, "")`
  - 当仓库中暂无全局映射时，回退到内置默认模型列表
- 价格信息：
  - 优先使用 `PricingService.GetModelPricing`
  - 无动态价格时回退到 `BillingService.GetModelPricing`
- 站点基础信息：
  - 使用 `SettingService.GetPublicSettings`

## 4. 后端实现策略

### 4.1 复用现有公开 SettingHandler

在 `SettingHandler` 中扩展一个新方法：

- `GetPublicModelPlaza`

优点：

- 不需要新增独立公开 handler 路由分组
- 能与 `GetPublicSettings` 共用公开设置上下文
- 可以直接返回站点名称、文档链接等信息

### 4.2 模型聚合逻辑

后端聚合时做三件事：

1. 整理原始模型 ID 列表
2. 基于模型名和定价元数据推断 provider / capabilities
3. 转换为前端可直接渲染的卡片数据

推断规则先以“可用、稳定、可扩展”为主，不追求一次性覆盖所有模型细节：

- `gpt-*` / `o*` 归类为 `openai`
- `claude-*` 归类为 `anthropic`
- `gemini-*` 或 `models/gemini-*` 归类为 `gemini`
- 其余优先使用 `litellm_provider`
- 仍无法识别时归类为 `other`

## 5. 前端实现策略

### 5.1 组件拆分

建议新增：

- `src/components/model-plaza/model-plaza-view.tsx`
- `src/lib/model-plaza-api.ts`

其中：

- `model-plaza-view.tsx` 负责筛选、统计、卡片展示
- `model-plaza-api.ts` 负责调用公开接口和定义类型

### 5.2 页面复用

- `src/app/models/page.tsx` 使用公开页壳子包裹模型广场主体
- `src/app/dashboard/models/page.tsx` 仅渲染模型广场主体

## 6. 第一阶段不做的内容

以下内容先不纳入本次提交：

- 模型详情页
- 后台可视化排序/推荐位配置
- 按模型实时测速和可用性探测
- 服务端分页
- 国际化文案全量接入
